# Scout Demo GIF Design Thinking

## Mental Model

The visitor is asking: "What happens when I give Scout a URL?" The demo should answer with a simple product-flow preview, not a full product tour. It should feel like watching a run ledger become usable artifacts.

## Information Architecture

| Content | Tier | Treatment |
|---|---|---|
| Demo headline | Primary | Compact launch-site section headline |
| Animated GIF | Primary | Large framed media asset |
| Three-step explanation | Secondary | Short labels beside or below media |
| Boundary note | Supporting | Plain-text caveat |

## Interaction Flow

The demo is passive media. The page action after watching should be either "Install locally" or "Open examples."

## Cognitive Load Budget

Visible chunks:

1. Demo headline.
2. Demo media.
3. Three-step explanation.
4. Boundary note.

This stays under the five-chunk budget.

## Emotional Journey

The demo should make Scout feel understandable in under a minute: URL in, evidence preserved, records exported. It should create confidence without pretending every site works.

## Design Pre-Mortem

- Risk: GIF implies certified live crawling for hard sites. Mitigation: use a neutral fictional category URL and explicit "No hard-site bypass guarantee."
- Risk: GIF feels like fake app UI. Mitigation: present it as a product-flow explainer, not a screenshot of the legacy `/app`.
- Risk: too much text inside the media. Mitigation: three frames, large labels, no tiny UI copy.

## Aesthetic Choice

Use the current Supadesign IndustrialGray launch-site language: black/ivory surfaces, evidence-ledger framing, and restrained industrial accents.
