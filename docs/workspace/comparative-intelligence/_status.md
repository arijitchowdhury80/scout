# Comparative Intelligence Status

Date: 2026-07-09
Status: Planning spike proposed

## Why This Exists

Beta feedback exposed a sharper commercial use case for Scout: users do not only
want raw website extraction, they want comparison-ready market answers.

The feature hypothesis is that Scout can turn a user-supplied entity, product,
service, location, or competitor set into a cited comparison matrix across
multiple websites.

## Current Decision

Treat this as a dedicated feature spike, not a launch-page copy tweak.

## Planning Artifacts

- `01-feedback-capture.md` - source feedback and product interpretation.
- `02-strategy.md` - positioning, target buyers, non-goals.
- `03-prd.md` - product requirements and acceptance criteria.
- `04-risk-and-compliance.md` - scraping, accuracy, and market-data risks.
- `05-technical-spike-plan.md` - staged build plan, tests, and architecture.
