## Emphasis Rules

- One hero per page or major section.
- Homepage sections must not compete with the hero.
- Playground is primary on the homepage, secondary/callout in docs.
- Docs page headings are primary wayfinding, not marketing claims.

## Component Constraints

- Use nav links for wayfinding; each label must map to the named section.
- Use tables for API reference.
- Use code blocks for install/API commands.
- Use compact callouts for playground links in docs.
- Avoid nested cards and decorative page-section cards.

## Responsive Requirements

- 375px: nav wraps cleanly, playground stacks, code blocks scroll.
- 768px: docs sections remain readable in one or two columns.
- 1024px: homepage order is obvious without crowding.
- 1280px+: full-width sections use existing Flux grid rhythm.

## Accessibility

- Links and controls keep visible focus.
- Playground tabs and buttons remain keyboard-accessible.
- Touch targets remain at least 44px.
- Do not rely on color alone for active nav state.
