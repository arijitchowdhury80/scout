# Scout Website UI/UX Constraints

Date: 2026-06-27

## Emphasis Rules

- One hero message only.
- No more than three primary CTAs in first viewport.
- Use real product imagery or screenshots.
- Do not make a generic gradient SaaS hero.
- Do not hide status/limitations.

## Component Constraints

- Buttons must be clear commands: Install, View Docs, Request Beta.
- Feature lists should be scannable cards or bands.
- Comparison table must be readable on mobile.
- Code snippets should be copy-friendly visually, even if static.

## Responsive Requirements

- 375px: single-column hero, image below text or background-cropped safely.
- 768px: two-column sections can appear.
- 1024px+: full feature grid.
- 1280px+: constrained content width; avoid ultra-wide text lines.

## Accessibility

- High text contrast.
- Visible focus states.
- Buttons/links with descriptive labels.
- Do not rely on color alone for status.

## Tone

- Operator-grade.
- Honest beta status.
- Strong but not boastful.
- Avoid "magic" and "unblock everything" language.
