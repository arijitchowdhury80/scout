# About Firecrawl
The API to search, scrape, and interact with the web at scale. 🔥 We turn the live internet into clean Markdown and structured data for LLMs, agents, and AI-native software.

## Why we exist
Firecrawl was built on a simple insight: large language models are powerful, but hard to feed well. Developers shouldn't have to duct-tape proxies, scrapers, orchestrators, and post-processing logic to give an LLM the data it needs. Today, Firecrawl is one of the fastest growing open source projects in the world.
The web holds nearly everything humans know — the most complete, real-time record of our knowledge. But this knowledge is trapped, scattered across millions of domains, locked behind JavaScript, and constantly changing. AI needs this data to be useful, to answer questions accurately, to take actions confidently, and to understand the world as it exists right now.
Firecrawl changes that. **We provide the infrastructure to make the web fully LLM-ready: structured, searchable, and usable in production.** We're building the eyes for AGI.
[ OUR MISSION ]
Our mission is to make web data work for AI — cleanly, scalably, and in real time.

## What we make possible
One API to turn the web into structured, LLM-ready data. No boilerplate, no infrastructure — just the building blocks for AI-native software.
[ 01 ]
###  [Search](https://docs.firecrawl.dev/features/search)The web as a database
Search the web and get full page content from the results — not just links. Feed real-time context and fresh knowledge into your agents and RAG pipelines.
[ 02 ]
###  [Scrape](https://docs.firecrawl.dev/features/scrape)Any page, any format
Convert any URL into clean markdown, structured JSON, screenshots, or semantic text. Describe what data you need in natural language and get back structured results — from any page, at any scale.
[ 03 ]
###  [Interact](https://docs.firecrawl.dev/features/interact)Cloud sandboxes for agents
Take actions on any page — click buttons, fill forms, and navigate dynamic content. Describe what you want in natural language, or write code for full control.
We're not a toolkit. We're the foundation for building AI-native software — where intelligence isn't an add-on, it's part of the core.
