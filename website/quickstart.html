Path 01 · Hosted beta

Call the live Scout API.

Hosted HTTP beta starts through /beta.html and is always metered. Email-only registration is the beta signup path. Do not use localhost for hosted calls. Hosted credits are capped and named plainly — Free is 5,000 credits, Monthly is 50,000 credits — never sold as unmetered or open-ended hosted crawling. 

export SCOUT_HOSTED_BASE_URL="https://scout.chowmes.com"
export SCOUT_HOSTED_API_KEY="paste-your-delivered-key"

curl "$SCOUT_HOSTED_BASE_URL/v1/hosted/me" \
  -H "Authorization: Bearer $SCOUT_HOSTED_API_KEY"

curl -X POST "$SCOUT_HOSTED_BASE_URL/v1/hosted/scrape" \
  -H "Authorization: Bearer $SCOUT_HOSTED_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com","formats":["markdown"],"timeout_ms":30000}'

Use the real domain

Hosted calls go to https://scout.chowmes.com/v1/hosted/* with Authorization: Bearer ....

Credits

/v1/hosted/me shows your plan, balance, and current limits. A basic hosted scrape consumes one credit.

Keys

Beta onboarding uses /v1/hosted/beta-key. Scout emails the API key; never paste hosted API keys into frontend code, public repos, screenshots, tickets, or shared logs.

API reference

The endpoints testers actually need.

GET /v1/hosted/me Check hosted beta plan, credits, limits, and retention.
POST /v1/hosted/beta-key Register for a finite-credit beta tester API key by email.
POST /v1/hosted/scrape Acquire one public URL and return content plus evidence metadata.
GET /health Verify a local or hosted Scout service is alive.
POST /v1/hosted/products Acquire product records through the hosted beta surface.
POST /v1/hosted/run/{use_case} Run company, PRISM, investor, careers, news, social, and locations workflows.
GET /v1/hosted/runs List hosted run summaries owned by the caller's beta key.

3. Inspect evidence

Open records.json, source_pages.json, blocked_pages.json, and extraction_report.md.

Examples

Run small, inspectable workflows first.

Page to markdown

Capture one public page as readable text with source metadata.

mkdir -p ./scout-runs/example-page
scout scrape https://example.com \
  > ./scout-runs/example-page/scrape.json

Product category to records

Create product records plus blocked-page evidence for a small listing run.

scout products books \
  --site books.toscrape.com \
  --start-url https://books.toscrape.com/ \
  --output-dir ./scout-runs/books-products

Company intelligence packet

Produce a compact company packet with overview, key URLs, and citable source pages.

scout run company \
  --query Adobe \
  --mode auto \
  --workdir ./scout-runs/adobe-company

Artifact contract

Every useful run should leave evidence behind.

records.json

Structured records created from the run.

source_pages.json

Source registry with URLs, providers, timestamps, and hashes.

blocked_pages.json

Blocked, sparse, or failed pages with provider attempts and reasons where available.

extraction_report.md

A human-readable summary of what worked, what failed, and what should be reviewed next.
