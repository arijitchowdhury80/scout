Features

The crawler is only the first step.

Scout wraps acquisition with the pieces downstream AI, search, research, and demo workflows actually need.

01

Acquisition ladder

Crawl4AI, direct fetch, scout-browser, user-browser capture, saved evidence, and API adapters under one contract.

02

Citation-grade evidence

Source registry, hashes, provider status, snippets, blocked pages, screenshots, and field-level citations.

03

Typed records

Product, company, investor, career, job, news, docs, research, social, location, and website-quality records.

04

Algolia-ready exports

Prepare object IDs, names, prices, URLs, images, categories, variants, warnings, and export files.

05

Local and hosted

Run locally through CLI/HTTP/Docker/skill, or use hosted beta keys when you want managed convenience.

06

Portable artifacts

JSON, JSONL, CSV, SQLite, source pages, blocked evidence, validation, and reports for replay and audit.

What people use it for

Acquisition primitives for real workflows.

Product catalogs

Extract listing records, preserve source evidence, and prepare exports for Algolia, JSONL, CSV, SQLite, or spreadsheet adapters.

Competitive intelligence

Collect company, investor, careers, news, docs, and website-quality evidence into repeatable artifact folders.

Agent tools

Give local agents a controlled acquisition layer with provenance instead of raw, untraceable fetches.

Demo and search builds

Turn a prospect or customer website into inspectable product records that can feed Algolia demos or evaluation indexes.

Research archives

Save the raw evidence, normalized records, and extraction report so future runs can compare what changed.

Browser-assisted capture

Use captured DOM and screenshots when normal crawlers are blocked, without pretending every site can be bypassed.
