Scout Pricing

Free local beta. Metered hosted convenience.

The economic rule is simple: local runs use your compute, so local package can be free. Hosted runs use Scout workers, browsers, storage, LLM calls, security, and support, so hosted usage must be metered and priced from real unit economics.

What one credit means

Credits map to resource cost, not marketing magic.

Standard

Light acquisition

1 scrape = 1 standard credit. 1 returned crawl page = 1 standard credit. 1 product or intelligence record = 1 standard credit.

Artifacts

Screenshots cost more

1 screenshot = 3 standard credits because it uses rendered page capture, image generation, transfer, and storage.

Browser

Browser work is separate

1 browser render = 5 browser credits. 1 browser minute = 10 browser credits. Browser credits are not included in the first public pay-as-you-go package.

Margin

Default target is met

Estimated gross margin: 74.1%. Break-even: 17 packs/month against a $120 monthly fixed-cost assumption.

Assumption

Numbers are adjustable

This is the launch candidate, not a forever price. If browser, LLM, support, firewall, or hosting costs rise, package credits or prices must change.

Hosted economics

No unlimited hosted crawling.

Credits

Finite hosted credits

Every hosted run consumes variable resources: fetches, browser minutes, screenshots, storage, retries, security, and support. Hosted beta uses caps while unit economics are validated, and external LLM use is disabled by default.

Limits

No unlimited hosted crawling

A one-time pass cannot economically support millions of calls. Hosted Scout must meter usage from day one and price from real costs.

Local

Local stays the default

Use local Scout when you need private artifacts, user-browser capture, repeatable research, or low-cost experimentation.
