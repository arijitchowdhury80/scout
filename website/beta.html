Hosted beta registration

Register and receive your API key by email.

Register with your name and email. Scout provisions a finite-credit hosted beta key for the tester and emails the API key to the address you provide. If email delivery is not configured yet, Scout records the request and follows up after delivery is enabled. Public signup never shows raw keys in the browser.

Hosted beta registration emails a finite-credit API key when delivery is configured. Until then, Scout records your request for follow-up.

The beta trial includes 100 standard credits for 30 days. Scout does not show raw API keys in the browser. The setup and delivery paths let testers self-register with name and email while Scout records every request for metering and support.

Card-backed beta setup

Beta testers can also verify the full payment pipeline with a $0 Stripe setup-mode checkout. Scout collects a payment method, charges nothing, then emails the beta API key after the signed webhook provisions the account.

Card-backed beta setup stays disabled until Stripe Checkout, webhook signing, and email delivery are configured.

Use the same email you used for beta registration. Scout will never show an API key in the browser.

Scout cannot show an old key again. If an account exists for this inbox, Scout can email a replacement key and disable the previous one.

Support

Beta support is evidence-first.

Tester handoff packet

Send beta testers the docs/product/private-beta-tester-handoff.md packet before their first run. It explains hosted finite credits, skill usage, artifact inspection, feedback format, and secret-handling boundaries.

Report bugs

Use the private beta bug template with your surface, version, command or endpoint, run ID, and non-sensitive artifact path.

Request workflows

Use the private beta feature template for missing product, company intel, hosted API, skill, or documentation workflows.

Keep secrets out

Do not paste API keys, cookies, private customer data, paid-site content, or screenshots containing secrets into public feedback.

Security reports should not be filed as public issues. Follow Scout legal and security guidance and report vulnerabilities privately to the maintainer.
