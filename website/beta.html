Scout Private Beta

Beta paths

Hosted HTTP first. Skill when you want agents.

Hosted HTTP

Register your name and email, receive the hosted API key by email after account provisioning, call `https://scout.chowmes.com/v1/hosted/*`, and inspect finite credits from `/account`.

Get beta tester API key

Claude/Codex skill

Use the skill wrapper from an agent, with hosted HTTP doing the acquisition work and preserving evidence.

Limited credits, not unlimited crawling. Hosted beta is capped and metered while unit economics are validated, so tester activity cannot create an unbounded bill.

Paid packages and overage rules remain separate from this beta registration path and require founder approval before public launch.

Support

Beta support is evidence-first.

Tester handoff packet

Send approved testers the docs/product/private-beta-tester-handoff.md packet before their first run. It explains hosted finite credits, skill usage, artifact inspection, feedback format, and secret-handling boundaries.

Report bugs

Use the private beta bug template with your surface, version, command or endpoint, run ID, and non-sensitive artifact path.

Request workflows

Use the private beta feature template for missing product, company intel, hosted API, skill, or documentation workflows.

Keep secrets out

Do not paste API keys, cookies, private customer data, paid-site content, or screenshots containing secrets into public feedback.

Security reports should not be filed as public issues. Follow Scout legal and security guidance and report vulnerabilities privately to the maintainer.
