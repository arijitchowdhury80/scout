Beta paths

Local beta

Hosted beta checkout

Creates a Stripe checkout session for a limited hosted beta pass when payment is configured.

Hosted beta is capped and metered. It is not unlimited crawling.

Current beta pass limits: 2,000 standard credits, 100 browser credits, 7-day artifact retention, 100 pages per run, and 1 concurrent hosted run.

Before inviting testers

Run the launch readiness checker.

Scout ships a packaged readiness check so beta operators can tell the difference between controlled private beta readiness and public launch readiness. 

scout launch-readiness
scout launch-readiness --require-public

Support

Beta support is evidence-first.

Report bugs

Use the private beta bug template with your surface, version, command or endpoint, run ID, and non-sensitive artifact path.

Request workflows

Use the private beta feature template for missing product, company intel, hosted API, CLI, Docker, skill, or documentation workflows.

Keep secrets out

Do not paste API keys, cookies, private customer data, paid-site content, or screenshots containing secrets into public feedback.

Security reports should not be filed as public issues. Follow Scout legal and security guidance and report vulnerabilities privately to the maintainer.
